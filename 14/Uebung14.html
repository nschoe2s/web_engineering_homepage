Question 1/12: Was ist mit der Orthogonalität des Webs gemeint und was hat dies für positive Auswirkungen auf das Web?
Mit Orthogonalität des Webs ist die Unabhängigkeit der einzelnen Komponenten gemeint.
Mit Orthogonalität des Webs ist die Unabhängigkeit der einzelnen Komponenten gemeint. Sprich Html CSS usw sind nicht voneinander abhängig. Es erlaubt weiterhin eine Modularität die ein Ökosystem fördert in der jeder Pakete erstellen und auf open Source ebene verbessern kann.

Question 5/12: Was sind Media Querys und wofür braucht man diese ?
Media Querys sind dafür da interaktiv über eine HTML Seite SQL Querys an eine Datenbank zu schicken und angezeigt zu bekommen.
Media Querys benötigt man für Responsive Web Design. Bei diesem wird unterschieden zwischen Desktop First (Wobei auf die Minimale breite reagiert wird), Mobile first(Wobei auf die Maximale breite reagiert wird) Damit die Webseite auf den jeweiligen geräten garantiert richtig angezeigt wird.
Media Querys werden benötigt um bestimmte HTML Elemente in einem Document zu findenBsp.:var matches = document.querySelectorAll("p");damit alle paragraphen aus einem HTML Dokument gesucht werden.
Media Querys sind dafür da ein RWD(Responsive Web Design) zu schaffen, indem ab bestimmten maximalen breiten(Desktop first) oder bestimmten minimalen breiten(Mobile first) neue CSS Regeln, welche in einer solchen Media Query Definiert sind, wirken wodurch man das Design einer Website an verschiedene Bildschirmgrößen anpassen kann.
Question 6/12: Was sind die Unterschiede zwischen HTTP GET und POST?
Die Namen dienen nur zur Kennzeichnung und die Übermittlung ist bei beiden gleich.
Bei der GET-Methode werden Parameter in der URL übermittelt und bei POST über den Request-body. Die GET Parameterlänge ist zudem begrenzt und es ist nicht möglich Dateien über GET zu übermitteln. Zudem können Seiten mit POST nicht einfach erneut aufgerufen werden, da dann die Parameter fehlen.
POST Seiten können im Browser nicht direkt über eine URL aufgerufen werden und können nur über JavaScript benutzt werden um z.B. Daten im Hintergrund zu laden.
Bei POST stehen die Parameter in der URL und bei GET im Request-body.

Question 11/12: Warum sind die Begriffe Internet und World Wide Web nicht synonym verwendbar?
Die Begriffe beschreiben beide die gleiche Technologie. Apple nannte sie Internet und Microsoft nannte sie World Wide Web.
Das Internet wird militärisch eingesetzt. Das World Wide Web ist auch der zivilen Bevölkerung zugänglich.
Internet und World Wide Web beschreiben jeweils andere Konzepte.Während das Internet die weltweite Vernetzung von Computersystemen auf Basis des TCP/IP-Protokollstacks beschreibt (also eher eine Technologie), beschreibt das World Wide Web einen Informationsraum auf Basis der Technologien HTML, HTTP und URI, in dem der Zugriff auf verteilte Inhalte ermöglicht wird (also eher eine Anwendung).
Das Internet beschreibt ein Netzwerk, das World Wide Web ist ein Synonym für den Browser.
